## 매일 메일 질문 답변하기

> 💡 ****다중 서버 환경에서 세션 기반 인증 방식을 사용하는 경우 발생할 수 있는 문제점은 무엇인가요?****

세션 불일치 문제가 발생할 수 있음

[gray]*로드밸런서를 통해 트래픽이 분배될 때 A 서버의 세션 정보가 B 서버에서 유효하지 않은 상황*[/]


**해결 방안**

- **스티키 세션 방식**
  - 사용자 요청이 항상 사용자 세션 정보가 저장된 서버로 가도록 고정하는 방식

  - 단순하지만, 트래픽을 분배하는 측면에서 로드밸런싱 이점을 크게 갖기 어려움


- **세션 클러스터링 방식**
  - 특정 서버에 사용자 세션 정보가 생성될 때, 다른 서버로 정보를 복제하는 방식

  - 위의 문제를 해결할 순 있지만, 세션 정보를 중복 저장하여 불필요한 메모리 사용 및 세션 정보 복제 과정에서 예상치 못한 네트워크 문제가 발생할 경우 일시적으로 세션 정보 유실 가능성 있음


- **스토리지 분리 방식**
  - 세션 정보를 저장하는 공간을 외부로 분리하는 방식

  - 스티키 세션 방식과 세션 클러스터링 방식에서 발생하는 문제를 해결할 수 있음

  - 스토리지에 대한 단일 장애 지점이 문제가 될 수 있으며, 복제 지연으로 인한 일시적인 세션 유실 문제 또한 발생할 수 있음, 추가적인 리소스를 요구함



> 💡 ****함수 선언식과 함수 표현식의 차이점에 대해서 설명해주세요.****

함수 선언식은 호이스팅 되어 코드 어디서든 호출이 가능하지만, 함수 표현식은 변수 할당 후 사용할 수 있음


- **함수 선언식**
  - 이름이 있는 함수

  - 자바스크립트 엔진이 코드를 실행하기 전에 메모리에 로드 → 호이스팅 발생

  - 함수 선언식으로 정의된 함수는 코드 어디서든 호출 가능


- **함수 표현식**
  - 변수에 익명 함수를 할당

  - 호이스팅 되지 않으며 변수에 할당된 이후에만 호출 가능
